# Integrate HMM Regimes into Cointegration Pipeline

### Simplified MVP (start simple, extend later)

- Features per symbol: r_t (15m log-return), vol_1d (rolling std over 96 bars). Standardize with `StandardScaler` on `cointegration_data`. No 1w vol, no z1d initially.
- Model: GaussianHMM, n_states=2, covariance='diag', random_state=0, n_iter=200. No inner CV; fit on `cointegration_data` only.
- Quality checks: min state support ≥10% of bars, avg dwell time ≥24 bars. If fail, mark symbol as “no-regime” and skip filtering for it.
- Label mapping: order states by mean vol → tags {low, high}; store numeric + tags.
- Filtering policy: only 'all' (intersection) initially. No 'any' or 'k_of_n' yet.
- Integration: extend `test_baskets_cointegration_parallel(..., regime_labels=None, include_states=None, policy='all')`; when labels and include_states provided, mask rows per basket before Johansen; require T ≥ n_assets*10 post-mask, else skip basket. No fallback to unfiltered in MVP.
- Persistence: write labels Parquet with minimal metadata. Defer model joblib saving.
- Backtest: add optional masking in `optimize_zscore.py`; log coverage ratio; require min bars and min trades; compare baseline vs regime-filtered Sharpe/DD/winrate. No bootstrap yet.

### Phase 2 (toggle on when ready)

- Add features: |r_t|, vol_1w, z1d; outlier clipping.
- Model selection via inner chronological split (70/30) and BIC tie-break across n_states ∈ {2,3,4}.
- Extra checks: separability tests; rare-state merge or fallback.
- Policies 'any' and 'k_of_n'; configurable fallback to unfiltered.
- Persist models via joblib; weekly retrain option for live.
- Statistical validation: paired bootstrap on daily returns.
